# BrainDump Dashboard — Product Requirements Document
**Version:** 1.0  
**Scope:** Dashboard UI only (not MCP server, not backend API)  
**Status:** Active

---

## 1. What This Dashboard Is

A **daily companion app** — not an admin panel.

The user opens it in the morning to orient themselves, checks back a few times throughout the day, and occasionally sits down for a slower review of a specific part of their life. They do not use it to input data (that happens in Claude/Gemini/etc). They use it to *understand where they are*.

Two modes in one app:

| Mode | When | Feel |
|---|---|---|
| **Brief** | Morning open, midday check-in | Motion, energy, time-aware — like a live newspaper front page |
| **Journal** | Slower, deliberate review of Goals / Finance / People / Notes | Calm, spacious, reflective — like reading a personal ledger |

The dashboard never asks the user to do anything. It surfaces, it shows, it interprets. Acting happens in the AI app.

---

## 2. Jobs to Be Done

| Job | Frequency | Current state |
|---|---|---|
| "What's happening today / this week?" | Every open | Not answered |
| "Am I moving forward on what matters?" | Daily | Partially (goals list exists) |
| "How am I doing financially / fitness-wise?" | Weekly | Not answered — raw data only |
| "What's on my mind lately?" | Weekly | Notes exist but no pattern |
| "Anything I've asked to be tracked that needs attention?" | Daily | Follow-ups exist but buried |
| "Give me a read on my life right now" | Weekly | Not answered |

---

## 3. Information Architecture

```
BrainDump
├── Home          ← default landing, motion-mode
├── Goals         ← journal-mode
├── Finance       ← journal-mode
├── People        ← journal-mode
├── Journal       ← notes + mood, journal-mode
└── Search        ← global semantic search across everything
```

Follow-ups and offloaded tasks are **not separate views** — they surface inline inside Home and the relevant section. They don't get their own nav item; that creates anxiety, not clarity.

Skills and Facts are **not primary nav** — they are contextual panels accessible from People and the profile area. They're reference data, not things you visit.

## 4. View Specs

---

### 4.1 Home — Daily Brief

**Purpose:** Answer "what do I need to know right now?" in under 10 seconds.

**Layout:** Vertical card stack. Cards are time-aware and update throughout the day.

#### Card 1 — Today's Pulse
Always visible at the top. Pulls from:
- Today's date + greeting (time-aware: Good morning / afternoon / evening)
- Upcoming events in the next 7 days
- Any delegated tasks with check-in dates today or overdue
- Count of pending follow-ups

```
Good morning, Yash.

📅  No events this week.
❓  2 follow-ups waiting.
📋  1 task you asked me to track is due for check-in.
```

Tapping a line takes you to the relevant section inline — no page navigation.

#### Card 2 — Goal Momentum
Shows the 3 most recently active goals with a simple momentum indicator — not a percentage, a signal:

- 🟢 **Moving** — updated in the last 7 days
- 🟡 **Slowing** — not updated in 8–21 days  
- 🔴 **Stalled** — not updated in 21+ days

```
🎯  ₹10Cr before 30          🔴 Stalled — 30 days ago
🎯  Run a 5K                 🟡 Slowing — 12 days ago
🎯  Post to YouTube weekly   🟢 Moving — 2 days ago
```

No progress bars on Home. Bars live in the Goals journal view.

#### Card 3 — Insights Strip
A single rotating insight generated by the LLM, refreshed once daily. Not a paragraph — a one-liner with an optional expand.

```
💡  Your gold position (₹50K) has been active for 62 days — worth a review.
    [ See financial snapshot → ]
```

See Section 6 for the full Insights system spec.

#### Card 4 — Follow-ups
If any pending follow-ups exist, show them here — max 2. A "see all" link expands inline.

```
❓  Your fitness goal hasn't been updated in 3 weeks — any progress?
❓  You asked to track YouTube posting — it's been 10 days, any update?
```

These are read-only on the dashboard. To answer them, the user goes to their AI app.

#### Card 5 — Recent Notes
Last 2 mood-tagged notes. Just the content and an emoji for mood. No actions.

```
😰  "Feeling burnt out after the sprint review"    3 days ago
😊  "Great catch-up with Rahul, feeling re-energised"    1 week ago
```

---

**Home design rules:**
- No tables, no lists longer than 3 items
- Every card has a natural "→ see more" that goes to the journal view
- Cards reorder by time-relevance — stalled goals float up, resolved follow-ups drop off
- Subtle animations on load — cards slide in sequentially, not all at once
- Color is used sparingly: 🔴🟡🟢 for momentum only

---

### 4.2 Goals — Journal Mode

**Purpose:** Sit down and review progress across all goals. Slow, readable, honest.

**Layout:** One goal per card, sorted by momentum (stalled first).

Each goal card shows:
- Title + category badge
- Momentum indicator (🔴🟡🟢) with days-since-update
- Progress bar (only here, not on Home)
- Deadline countdown: "127 days left" or "Overdue by 3 days"
- Milestones if any — checked/unchecked list
- Last note stored against this goal

**Filter bar:** All · Active · Stalled (21d+) · Completed · Abandoned

**No add button.** Goals are created via the AI app.

---

### 4.3 Finance — Journal Mode

**Purpose:** Understand current financial position at a glance. Like reading a personal balance sheet.

**Layout:** Two sections.

**Section A — AI Snapshot** (top, prominent)
An LLM-generated paragraph — 3–4 sentences summarising the user's financial picture based on all stored financial facts and goals. Refreshed on-demand (button), cached for 24h.

```
You have ₹3L actively invested across stocks, FDs, and gold. Your gold 
position is 62 days old with no status update — worth checking if you're 
still holding. You have no emergency fund against a ₹1L target, and your 
₹10Cr goal is at 0% with no recent progress.
```

**Section B — Position Table**
Clean table. Columns: Type · Asset · Amount · Date · Age · Status

Age = days since transaction date, colour-coded:
- Under 30 days: slate
- 30–90 days: amber
- 90+ days: red (stale, should review)

**No charts for now.** Charts need enough data points to be meaningful. With 3–5 financial facts, a table is cleaner.

---

### 4.4 People — Journal Mode

**Purpose:** Keep mental context on relationships. Who are the people in your life and what do you know about them?

**Layout:** Card grid. Each card:
- Name + relationship type + icon
- Notes snippet (first 80 chars)
- Last mentioned date

Future (Phase 2+): interaction log, "you haven't mentioned X in 30 days."

---

### 4.5 Journal — Notes + Mood

**Purpose:** Reflective review. See emotional patterns, read back recent thoughts.

**Layout:** Two panels.

**Left — Mood Timeline** (if 5+ notes with mood tags exist)
A simple dot timeline: each note is a dot coloured by mood, plotted over the last 30 days. No axes labels, no numbers. Just the shape of emotional activity. If not enough data, show a placeholder: "Mood patterns will appear as you add more notes."

**Right — Note feed**
Chronological list of notes, newest first. Each note shows:
- Content
- Mood emoji (if tagged)
- Emotion tags as small pills
- Category badge
- Date

**Filter:** All · This week · This month · By mood

---

### 4.6 Search

Global semantic search across facts, notes, goals, events, financial facts, people, skills. Returns ranked results with a source type badge on each. Already built — just needs a dedicated view with proper result rendering (not just the memories view).

## 5. Navigation Design

**Top nav** — horizontal, persistent:

```
🧠 BrainDump   [Home]  [Goals]  [Finance]  [People]  [Journal]  [🔍]   [Profile]
```

- Active view is underlined, not filled (lighter than current)
- Goal momentum indicator bleeds into the nav: Goals tab shows a red dot if any goal is stalled 21+ days
- Follow-up count badge on Home if any pending

**No sidebar.** Single-level navigation only.

**Profile** — a slide-in panel from the right, not a separate view. Shows stored facts, user profile fields, and a "copy context" button that generates the `get_context()` text output for pasting into any AI.

## 6. Insights System

### What it is
LLM-generated synthesis of stored data into human-readable signals. The only place in the product where an LLM runs on the backend.

### What it is NOT
It does not store new data. It does not make decisions. It does not replace the user's judgment. It reads what's there and says what it notices.

### Technical approach
- **Model:** Claude Haiku (fast, cheap — this is reading data not reasoning)
- **Trigger:** On-demand button ("Refresh insights") + automatic once on first daily open
- **Cache:** 24 hours. Stored in a new `insights` table with `category`, `content`, `generated_at`
- **Input:** Structured output from `get_context()` — the same data the AI apps already use
- **Output:** Per-category insight, 1–3 sentences max

### Insight categories

| Category | Example output |
|---|---|
| Financial | "Your gold position is 62 days old with no status update. Your emergency fund is ₹0 of your ₹1L target — no active savings goal to address this." |
| Health & Fitness | "Your 5K goal has 0% progress and hasn't been updated in 30 days. Your last health note mentioned not being able to run the full distance." |
| Goals overall | "3 of your 4 goals are stalled. The most active is your YouTube goal, updated 2 days ago." |
| Mood pattern | "Your last 5 notes skew stressed or anxious. Most are tagged to work/career contexts." |
| Career | "No career-related updates in the past 2 weeks. You have one active career goal with no milestones set." |

### API key handling
`ANTHROPIC_API_KEY` read from environment. If not set, the Insights section shows a "connect your API key to enable insights" prompt — the rest of the dashboard works fully without it.

### Privacy note (for Phase 2)
Insights are generated server-side. In Phase 2 (multi-user SaaS), the API call goes from the user's own server instance, not through a shared BrainDump backend. User data never transits through BrainDump infrastructure.

## 7. Design Principles

1. **No empty states that feel like failure.** If a section has no data, don't show a sad illustration. Show: "Nothing here yet — tell Claude about your finances and it'll appear here."

2. **Time-awareness everywhere.** "3 weeks ago" not "2026-05-09". Relative dates on Home, absolute on journal views where precision matters (Financial table).

3. **Motion on Home, stillness elsewhere.** The Home view uses subtle animation — cards animate in, the insights strip rotates. All other views are static. No decorative motion in journal mode.

4. **Colour is signal, not decoration.** 🔴🟡🟢 for goal momentum. Age-colour on financial rows. Mood colour on the journal timeline. Nowhere else.

5. **Nothing requires user input.** No forms, no save buttons, no text fields (except the search bar and the follow-up answer box which we're removing from the dashboard — answers happen in the AI app).

6. **The dashboard doesn't judge.** Stalled goals are shown as stalled, not flagged as failure. Stressed mood entries are shown, not highlighted as bad. The insight language is observational, not prescriptive.

---

## 8. What We Are NOT Building

- **Add/edit forms** — data input happens in AI apps via MCP
- **Follow-up answer UI** — answering happens in the AI app, not the dashboard
- **Charts before we have data** — no pie charts, bar charts, or trend lines until Phase 2 when multi-user data density justifies them
- **Notifications or push alerts** — the dashboard is pull, not push
- **Mobile layout** — Phase 4 per main PRD; design for desktop only now
